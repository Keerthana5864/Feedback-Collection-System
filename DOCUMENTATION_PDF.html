📌 Project Overview

The Faculty Feedback System is a full-stack web application developed for your institution that allows students to submit feedback about faculty members. Administrators can view, analyse, and respond to feedback through a rich admin dashboard.

Key Objectives

  • Enable students to rate faculty on a 1–5 star scale with comments
  • Provide administrators a real-time analytics dashboard
  • Instant low-rating alerts to detect problematic patterns early
  • Secure role-based access for students and admins
  • OTP-based password reset via phone + email verification

Technology Stack

Layer Technology
Frontend HTML5, CSS3, Vanilla JavaScript, Chart.js
Backend Python 3, Flask, Flask-CORS, python-dotenv
Database Supabase (PostgreSQL) — cloud hosted

Click-to-Navigate Feature

Clicking any notification item automatically switches to the Detailed Logs tab, smoothly scrolls to the exact feedback row, and highlights it with a yellow flash animation for easy identification.

🔗 API Endpoints & Authentication

REST API Endpoints (http://localhost:5000)

Method Endpoint Description
POST /api/login Authenticate user, return role
POST /api/register Create new student/admin account
GET /api/stats All feedback + aggregate stats (admin)
POST /api/feedback Submit new feedback (students)
POST /api/admin-response Admin submits reply to feedback
POST /api/forgot-password-phone Verify email + phone, issue OTP
POST /api/reset-password-phone Reset password using phone OTP

Password Reset — 3-Step OTP Flow

1
Identity Verification

Student enters Email + Phone. Backend verifies email in users table and phone in feedback table.

2
OTP Entry

6-digit OTP generated server-side and stored in memory. Displayed on screen for demo purposes.

3
New Password

Student sets new password. Updated in users table by email. OTP cleared from memory.

▶️ How to Run & Future Plans

Setup Steps

1
Supabase Setup

Create a project at supabase.com → Run schema.sql in SQL Editor → Copy URL & API key.

2
Configure .env

Edit backend/.env with your SUPABASE_URL and SUPABASE_KEY values.

3
Install & Run Backend 
cd backend
pip install -r requirements.txt
python app.py
4
Open Frontend

Open frontend/index.html in a browser — or serve via python -m http.server 8080.
